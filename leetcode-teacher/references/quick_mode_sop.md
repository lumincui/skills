# 快速模式 SOP

出题 → 对话引导思考 → 复盘讲解

无脚手架、无代码书写、无测试验证。适合复习或口头推导场景。

## Phase 1：出题

**目标**：选取一道适合用户当前状态的题目。

**操作步骤**：

1. 读取 leetcode.json，检查 progress 和 study_plan

2. 根据优先级选取题目：
   ```
   待复习/今日计划 > 未做过 > 已通过（跳过）
   ```

3. 展示题目：
   - 题号、题目名、难度标签、分类
   - 题目描述、示例、约束

---

## Phase 2：引导思考

**目标**：通过对话问答引导用户思考算法思路。

**Why**：让用户在对话中自己推导出来，比直接告知记得更牢。

**操作步骤**：

1. **分析题目**：
   - 这道题考核的核心能力是什么？
   - 常见实现错误通常在哪？
   - 有哪些执行分支？

2. **引导性问题**（根据题目类型选择）：

   | 题型 | 引导方向 |
   |------|---------|
   | DP | 状态如何定义？转移方程是什么？ base case 是什么？ |
   | 双指针 | 指针如何移动？什么时候相遇？ |
   | 滑动窗口 | 窗口如何扩张和收缩？维护什么状态？ |
   | 回溯 | 如何做选择和撤销？剪枝条件是什么？ |
   | 二分 | 如何判断目标在哪半边？边界如何更新？ |
   | 链表 | 需要几个指针？ fast-slow 的相遇点如何利用？ |
   | 树 | 遍历顺序是什么？返回值表示什么？ |

3. **对话节奏**：
   - 先问方向："你觉得用什么算法？"
   - 再问细节："状态怎么定义？"
   - 最后问边界："base case 是什么？"
   - 用户答对 → 肯定并推进
   - 用户答错 → 追问而非直接纠正

4. **约束**：只给引导性问题，不给答案，不给代码

---

## Phase 3：复盘讲解

**目标**：确认用户理解，巩固算法思维。

**触发**：用户请求"讲解"或"给答案"，或表示已经想清楚。

**操作步骤**：

1. **确认用户当前思路**：
   - "你的解法是什么？"
   - "时间复杂度是多少？"

2. **讲解**：
   - 给出最优解代码（带注释）
   - 对比用户思路与最优解
   - 分析时间/空间复杂度
   - 重点解释"为什么这个优化能成立"

3. **讲解后**：
   - 询问用户是否理解
   - 根据反馈更新学习计划

---

## Phase 4：更新进度

**操作步骤**：

1. 执行 `python3 scripts/update_progress.py <题号> need_review`
   - 快速模式不做代码验证，标记 need_review 便于之后普通模式复习

2. 判断后续：
   - 未达到每日目标 → 询问是否继续下一题
   - 已达到每日目标 → 建议收工

---

## Phase 5：收工

**目标**：整理当日学习成果。

**触发**：用户说"收工"、"完成"、"结束了"。

**操作步骤**：

1. Git 提交推送（如果有 git 仓库）
2. 检查学习计划
3. 告知用户今日总结
