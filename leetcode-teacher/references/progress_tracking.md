# 进度追踪格式参考

本文档定义 README.md 中的进度表格和学习计划格式。

## 进度表格格式

```markdown
## 学习进度

| 题号 | 题目名 | 类型 | 状态 | 完成日期 |
|------|--------|------|------|----------|
| 1 | Two Sum | array | ✅ 通过 | 2024-01-15 |
| 2 | Add Two Numbers | linked_list | 🔄 需复习 | 2024-01-14 |
```

### 状态说明

- `✅ 通过`：用户独立完成所有代码并测试通过
- `🔄 需复习`：用户请求了答案、提示或未能完全独立完成

---

## 学习计划章节格式

```markdown
## 学习计划

### 待复习
- <题号> <题目名> - <原因> - <计划>

### 今日待做
- <题号> <题目名> - <原因>

### 知识点备忘
- <概念> - <理解要点>
```

### 使用规则

1. **待复习**：当用户说"明天做XX题"或"待复习XX题"时更新
2. **今日待做**：根据用户目标动态维护
3. **知识点备忘**：用户掌握某个知识点时，询问是否移入

---

## 自动序列触发条件

### 每日三题完成

当 README 中完成日期为今天的题目达到 3 道时：

1. 告知用户"今日三题目标已完成！"
2. Git 提交推送（如果不是 git 仓库则跳过）
3. 更新 Todoist：搜索今日到期任务，添加评论，标记完成
4. 检查学习计划

### 收工序列

当用户说 `收工` / `完成` / `结束了` 时：

1. Git 提交推送
2. 更新 Todoist（如有匹配任务）
3. 检查学习计划，确认今日待做是否完成
