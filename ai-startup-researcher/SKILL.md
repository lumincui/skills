# AI初创公司信息收集器

这个skill帮助你系统性地收集、整理和归纳AI初创公司的信息，并按照知识库的格式进行存储。

## 何时使用

在以下情况下主动使用此skill：
- 用户明确要求收集或整理某个AI公司的信息
- 用户讨论某个AI初创公司（即使只是提到名字）
- 用户想了解某个公司的融资、团队、产品等信息
- 用户在做AI行业研究或竞品分析

## 工作流程

### 第一步：确认公司和收集方向

首先确认用户想研究的目标公司，然后确定信息收集的重点方向：
- 如果是完整研究：收集所有维度的信息
- 如果是特定方向：聚焦用户关心的领域（如融资、产品、团队等）

### 第二步：系统性搜索信息

#### 分阶段搜索策略

**阶段 1：基础信息收集**（优先执行）
- 官网 + 官方博客
- 36氪、晚点LatePost、TechCrunch 最新报道
- Crunchbase/IT桔子融资信息
- LinkedIn 公司页面

**阶段 2：创始人信息深挖**
- 创始人 LinkedIn 个人页面
- 创始人 Twitter/X 账号
- 创始人姓名 + 学校/前公司搜索
- "创始人姓名 + interview" / "创始人姓名 + 访谈"
- 百度百科/维基百科（如果有）

**阶段 3：产品和用户信息**
- Product Hunt 发布页面
- GitHub（如果是技术产品）
- 用户评测：知乎、即刻、Reddit、Twitter
- "产品名 + review" / "产品名 + 测评"
- App Store/Google Play 评论（如果是 App）

**阶段 4：行业和竞争分析**
- 行业媒体深度报道
- 竞品对比文章
- 投资机构行业分析
- 技术博客（如果有）

**阶段 5：补充和验证**
- 交叉验证关键数据
- 补充缺失信息
- 查找最新动态

#### 高价值信息来源

**中文来源**（按优先级）：
1. **综合媒体**：36氪、晚点LatePost、虎嗅、钛媒体
2. **AI 垂直媒体**：机器之心、新智元、量子位、AI 科技评论、智东西
3. **融资平台**：IT桔子、烯牛数据、天眼查、企查查
4. **百科类**：百度百科、维基百科
5. **社交媒体**：微信公众号、即刻、知乎、V2EX
6. **新闻门户**：腾讯新闻、搜狐科技、网易科技

**英文来源**（按优先级）：
1. **综合媒体**：TechCrunch、The Information、Reuters、Bloomberg
2. **AI 垂直媒体**：VentureBeat AI、The Verge AI、MIT Technology Review
3. **融资平台**：Crunchbase、PitchBook
4. **社交媒体**：Twitter/X、LinkedIn、Reddit（r/MachineLearning）
5. **产品平台**：Product Hunt、Hacker News
6. **技术社区**：GitHub、Stack Overflow

**特定信息最佳来源**：
- **创始人背景**：LinkedIn、Twitter、百度百科、维基百科
- **融资详情**：Crunchbase、IT桔子、投资方官网
- **产品信息**：官网、Product Hunt、用户评测
- **用户数据**：财报、媒体报道、第三方统计
- **技术细节**：GitHub、技术博客、论文
- **行业分析**：投资机构报告、行业媒体深度报道

#### 搜索技巧

**关键词组合**：
- 基础："公司名+融资"、"公司名+创始人"、"公司名+产品"
- 创始人："创始人姓名 + 公司名"、"创始人姓名 + LinkedIn"、"创始人姓名 + 背景"
- 产品："产品名 + review"、"产品名 + 评测"、"产品名 + Product Hunt"
- 数据："公司名 + 用户数"、"公司名 + ARR"、"公司名 + 收入"
- 融资："公司名 + 融资细节"、"公司名 + 投资方"、"投资方 + 公司名"

**搜索运算符**：
- `site:linkedin.com "创始人姓名"` - 限制在 LinkedIn 搜索
- `site:36kr.com "公司名"` - 限制在 36氪 搜索
- `"Flowith AI" founder` - 精确匹配
- `"公司名" 融资 OR 投资 OR funding` - 多关键词

**时间限制**：
- Google 搜索工具：限制在过去一年/一个月
- 添加年份关键词："公司名 2025"、"公司名 2024"

**多语言搜索**：
- 同时搜索中英文名称
- 使用翻译工具查找本地化名称
- 搜索不同语言的用户评价

### 第二步半：信息质量检查与深度搜索

**关键步骤**：在完成初步搜索后，必须评估信息完整度，决定是否需要深度搜索。

#### 信息完整度评估

检查是否收集到以下核心信息：

**必填信息清单**（至少需要 5/7）：
- [ ] 创始人完整姓名（中英文）
- [ ] 创始人教育背景或工作经历
- [ ] 具体融资金额（美元）
- [ ] 团队规模（具体数字或范围）
- [ ] 公司成立时间
- [ ] 总部地点
- [ ] 核心产品功能

**高价值信息清单**（优先收集）：
- [ ] 用户数据（MAU/总用户）
- [ ] 收入数据（ARR/收入）
- [ ] 盈利状态
- [ ] 创始人完整履历
- [ ] 产品详细功能和特点
- [ ] 竞争优势
- [ ] 重要里程碑

#### 触发深度搜索的条件

**满足以下任意 2 条，必须进行深度搜索**：
1. 创始人信息少于 2 项（如只有姓名，无背景）
2. 团队规模未知或仅知道"待补充"
3. 用户数据完全缺失
4. 收入数据完全缺失
5. 产品信息过于简略（少于 3 个功能点）
6. 必填信息完整度 < 70%
7. 关键数据来源单一（只有一个来源）

#### 深度搜索策略

**1. 创始人信息深挖**：
- 搜索创始人姓名的所有变体（中文、英文、昵称）
- 查找创始人的所有社交媒体（LinkedIn、Twitter、即刻、知乎）
- 搜索创始人之前的创业项目
- 搜索"创始人姓名 + 访谈"、"创始人姓名 + 演讲"
- 查找创始人母校的校友信息
- 搜索投资方对创始人的评价

**2. 产品和数据信息深挖**：
- 搜索 Product Hunt 发布页面和评论
- 查找第三方评测和对比文章
- 搜索用户在社交媒体的讨论
- 查找技术博客和开发日志
- 搜索 GitHub 仓库（如果有）
- 查找招聘信息透露的团队规模
- 搜索应用商店评论和评分

**3. 融资信息深挖**：
- 搜索投资方的官方公告和投资组合
- 查找投资方的媒体报道
- 搜索融资平台的详细数据
- 查找投资者对该项目的公开评论
- 搜索同一赛道其他公司的融资情况作为参考

**4. 垂直领域深挖**：
- AI 领域：机器之心、新智元、量子位、AI 科技评论
- 特定赛道：搜索 "{赛道} + 公司名"（如 "AI Agent + Flowith"）
- 查找行业报告和榜单
- 搜索学术会议和论文（如果是技术驱动公司）

**5. 交叉验证**：
- 对关键数据使用至少 2 个独立来源验证
- 对比不同来源的数据差异
- 优先使用官方披露和权威媒体报道
- 标注数据的可信度（✅ ⚠️ ❓）

#### 深度搜索执行流程

1. **识别信息缺口**：列出所有缺失的核心信息
2. **针对性搜索**：对每个缺口执行专门的搜索策略
3. **使用替代信息源**：
   - 如果没有官方数据，使用媒体报道的估算
   - 如果没有精确数字，使用范围（如"10-50人"）
   - 明确标注信息来源和可信度
4. **标记待补充信息**：
   - `[待补充]` - 信息存在但未找到
   - `[未披露]` - 信息可能未公开
   - `[推测]` - 基于其他信息推测
5. **持续迭代**：如果仍然信息不足，考虑：
   - 搜索创始人的个人博客或技术文章
   - 查找公司员工的社交媒体
   - 搜索投资者对该赛道的整体分析
   - 等待一段时间后复查（可能新信息出现）

## 高级搜索技巧

### 社交媒体搜索

**LinkedIn**：
- 搜索创始人姓名 + 公司名组合
- 查看创始人的工作经历和教育背景
- 查看公司的员工数量和招聘信息
- 查看共同联系人（可能提供更多线索）

**Twitter/X**：
- 使用高级搜索：`from:用户名 关键词`
- 搜索公司官方账号的历史推文
- 查找创始人发布的关于产品的推文
- 搜索用户对产品的评价和讨论

**Reddit**：
- `site:reddit.com "公司名"`
- 搜索相关子版块：r/MachineLearning, r/artificial, r/startups
- 查找用户的真实使用体验

**中文社区**：
- **即刻**：搜索公司名和产品名，查找用户讨论
- **知乎**：搜索相关问题和回答
- **V2EX**：查找技术社区的讨论
- **微信公众号**：使用搜狗微信搜索

### 技术信息搜索

**GitHub**：
- 搜索公司名或产品名
- 查看代码仓库的 star 数和 fork 数
- 查看贡献者信息
- 查看 issues 和 discussions

**技术博客**：
- 搜索"公司名 + 技术博客"
- 搜索"公司名 + engineering blog"
- 查找技术文章和架构分享

**学术资源**：
- Google Scholar：搜索创始人姓名
- arXiv：查找相关论文
- 会议论文：NeurIPS、ICML、CVPR 等

### 数据验证技巧

**交叉验证方法**：
1. **融资数据**：
   - 对比 Crunchbase、IT桔子、媒体报道
   - 查找投资方的官方公告
   - 注意区分"累计融资"和"单轮融资"

2. **用户数据**：
   - 对比官方披露和媒体报道
   - 查找第三方统计平台（SimilarWeb 等）
   - 注意区分"注册用户"、"月活"、"日活"

3. **团队规模**：
   - LinkedIn 公司页面显示的员工数
   - 招聘信息透露的信息
   - 官网团队介绍
   - 媒体报道中的数字

4. **收入数据**：
   - 官方披露（如果是上市公司）
   - 投资者透露的信息
   - 行业分析师的估算
   - 注意区分"收入"、"ARR"、"GMV"

## 信息不足处理流程

### 识别和处理信息缺口

**步骤 1：创建信息缺口清单**
```
信息缺口清单：
- [ ] 创始人完整履历（目前只有姓名）
- [ ] 团队规模（目前标注"待补充"）
- [ ] 用户数据（无任何数据）
- [ ] 收入数据（无任何数据）
- [ ] 产品详细功能（只有简短描述）
```

**步骤 2：针对性补充搜索**

对每个缺口执行专门搜索：

- **创始人信息缺失** →
  - 搜索所有姓名变体
  - 查找所有社交媒体
  - 搜索之前的创业项目
  - 查找母校校友信息

- **数据缺失** →
  - 查找财报（上市公司）
  - 搜索媒体报道中的数据引用
  - 查找第三方统计平台
  - 搜索投资者公开评论

- **产品信息不足** →
  - Product Hunt 页面
  - 用户评测和对比文章
  - 官方文档和教程
  - GitHub 仓库

**步骤 3：使用替代信息源**

当无法找到精确信息时：

| 缺失信息 | 替代方案 | 标注方式 |
|---------|---------|---------|
| 精确团队规模 | 使用范围（如"10-50人"） | `10-50人 ❓` |
| 用户数据 | 使用"快速增长"、"数万用户"等描述 | `数万用户（媒体报道）⚠️` |
| 收入数据 | 使用融资情况推测发展阶段 | `[未披露]` |
| 成立时间 | 使用融资时间推测（通常融资前 1-2 年成立） | `2024年左右（推测）❓` |

**步骤 4：明确标注信息状态**

使用统一标记系统：
- `✅` - 多源验证或官方披露
- `⚠️` - 单一来源或需谨慎
- `❓` - 推测或计算值
- `[待补充]` - 信息存在但未找到
- `[未披露]` - 信息可能未公开
- `[推测]` - 基于其他信息推测

**步骤 5：建立持续更新机制**

在报告末尾添加：
```markdown
## 待补充信息

- [ ] 创始人详细履历
- [ ] 精确团队规模
- [ ] 用户增长数据
- [ ] 收入和盈利情况

**最后更新时间**：{YYYY-MM-DD}
**信息来源**：{主要来源列表}
**下次建议复查**：{YYYY-MM-DD}（通常 3 个月后）
```

### 信息不足时的报告策略

**策略 1：透明呈现**
- 明确标注哪些信息缺失
- 不要编造或猜测数据
- 解释为什么某些信息难以获取

**策略 2：提供上下文**
- 即使没有精确数据，也提供相关背景
- 例如："虽然用户数未披露，但产品在 Product Hunt 排名第一，显示市场认可度较高"

**策略 3：使用对比**
- 与同类公司的信息进行对比
- 提供行业平均水平作为参考
- 例如："团队规模未披露，但同类种子轮公司通常为 5-20 人"

**策略 4：建议后续行动**
- 列出需要持续关注的信息
- 建议用户如何获取最新信息
- 提供可能的官方渠道链接



### 数据可信度标记

在报告中使用以下标记标注数据来源可靠性：

| 标记 | 含义 | 使用场景 |
|------|------|----------|

## 注意事项

1. **数据准确性**：
   - 优先使用官方数据和多源验证的信息
   - 对于不确定的数据，明确标注为"待确认"或"推测"
   - 注意区分事实和观点
   - **重要**：不要因为信息不足就放弃搜索，使用深度搜索策略

2. **时效性**：
   - 优先收集最新信息（6-12个月内）
   - 在报告中注明信息的时间点
   - 定期更新已有公司的信息
   - **重要**：注意信息的发布时间，优先使用最近的报道

3. **完整性**：
   - 不强制要求所有字段都有数据
   - 缺失信息可以标注"待补充"
   - 根据可用信息灵活调整报告结构
   - **重要**：当必填信息完整度 < 70% 时，必须进行深度搜索

4. **客观性**：
   - 呈现事实，避免主观评价
   - 平衡展示优势和风险
   - 引用多个来源，避免单一视角
   - **重要**：包含用户反馈时，要平衡正面和负面评价

5. **搜索效率**：
   - 先执行基础搜索，再根据信息完整度决定是否深度搜索
   - 使用分阶段搜索策略，避免一开始就过度搜索
   - 对高价值信息（创始人、融资、用户数据）投入更多搜索时间
   - **重要**：记录搜索过程，方便后续复查和补充

6. **信息验证**：
   - 对关键数据（融资金额、用户数、团队规模）使用至少 2 个独立来源验证
   - 注意不同来源的数据可能存在差异
   - 优先使用官方披露 > 权威媒体 > 行业媒体 > 社交媒体
   - **重要**：发现数据矛盾时，明确标注并说明来源

7. **特殊注意事项**：
   - **新成立公司**：信息可能较少，重点收集创始人和融资信息
   - **海外公司**：注意使用英文名称搜索，关注海外媒体
   - **隐形模式公司**：尊重未公开信息，不要过度挖掘
   - **被收购公司**：注意区分被收购前后的信息
   - **已倒闭公司**：标注状态，收集失败原因分析

## 经验总结

### 典型案例：Flowith 调研

**初始状态**：
- 第一次搜索只获得基本融资信息
- 创始人背景、产品详情、用户数据都缺失
- 信息完整度约 30%

**深度搜索后**：
- 获得完整创始人信息（3位创始人的详细履历）
- 获得完整产品矩阵（5个产品）
- 获得关键数据（ARR 130万美元，MAU 8万）
- 获得团队信息（10人，95后+00后）
- 信息完整度提升至 90%+

**关键经验**：

1. **不要满足于基础信息**：
   - 当发现创始人只有"三位创始人"这样模糊的信息时，必须深挖
   - 使用创始人姓名的各种变体进行搜索
   - 查找创始人的社交媒体和过往经历

2. **善用中文媒体**：
   - 百度百科可能包含详细信息
   - 腾讯新闻、搜狐科技等门户媒体有深度报道
   - AI 垂直媒体（机器之心、新智元等）有专业分析
   - 投资方公众号会发布融资公告

3. **交叉验证是关键**：
   - 融资金额：对比多个融资平台和媒体报道
   - 团队规模：对比 LinkedIn、招聘信息、媒体报道
   - 用户数据：对比官方披露和第三方统计

4. **产品信息要详细**：
   - 不仅要知道产品是什么，还要知道产品矩阵
   - 查找每个产品的发布时间和特点
   - 收集用户评价和行业认可（如 Product Hunt 排名）

5. **数据可信度标记很重要**：
   - 使用 ✅ ⚠️ ❓ 标记让读者了解信息可靠性
   - 明确标注信息来源
   - 区分官方披露和媒体报道

### 常见信息缺口及解决方案

| 信息缺口 | 常见原因 | 解决方案 |
|---------|---------|---------|
| 创始人背景缺失 | 公司低调，创始人无公开社交媒体 | 搜索投资方公告，查找母校校友信息 |
| 团队规模未知 | 初创公司不愿披露 | LinkedIn 查看员工数，搜索招聘信息 |
| 用户数据缺失 | 未到披露阶段 | 查找第三方统计，使用"快速增长"等描述 |
| 收入数据缺失 | 私密公司 | 查找投资者评论，使用融资情况推测阶段 |
| 产品功能不详 | 官网信息简略 | Product Hunt、用户评测、技术博客 |
