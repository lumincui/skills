# 论文类文档指引

面向学术论文的阅读笔记：帮助读者快速抓住一篇论文的核心论点、理解其完整论证过程、评估研究贡献与局限。

## 核心目标

论文笔记不是摘要的翻译，而是对论文智识结构的重建：

- **核心论点**：作者声称证明了什么？（一句话）
- **论证过程**：作者如何一步步支撑这个论点？
- **研究贡献**：这篇论文在领域中推进了什么？
- **局限与反驳**：论证在哪里存在漏洞或边界？

## 文档结构

### 推荐结构

```
## 核心论点

## 论证过程

## 研究贡献

## 局限与反驳

## 相关工作

## 参考
```

## 各节写法

### 核心论点

用**一到两句话**，直接说明论文的中心主张。

- 这不是"研究了什么话题"，而是"证明了/发现了/提出了什么具体结论"
- 好的核心论点是可以被质疑或证伪的命题，不是模糊的描述
- 如果论文本身有 abstract 中的 contribution 句，可以从那里提炼，但要去掉冗余，只保留最核心的主张

**区分方式：**

| 错误写法（话题） | 正确写法（论点） |
|-----------------|-----------------|
| 研究了大语言模型的推理能力 | 思维链提示在算术推理上的准确率比标准提示高 40%，且优势在模型规模超过 100B 时显著出现 |
| 提出了一种新的注意力机制 | 线性注意力在保持 O(n) 复杂度的同时，在长序列任务上与标准注意力性能相当 |

---

### 论证过程

按**逻辑推进顺序**（不是论文章节顺序）还原作者的论证链。

每个论证步骤包含：

1. **这一步的子论点**：作者在这里试图证明什么
2. **使用的方法或证据**：实验设计、数据集、理论推导、对照组
3. **得到的结论**：这一步的结果是什么

**格式示例：**

```
**步骤 1：建立基线**
- 子论点：标准提示在复杂推理任务上存在明显上限
- 方法：在 GSM8K 等 5 个数学数据集上测试 GPT-3（175B）的 zero-shot 准确率
- 结论：准确率在 17%–28% 之间，验证了改进空间的存在

**步骤 2：引入干预**
- 子论点：加入中间推理步骤能显著提升准确率
- 方法：用 8 个少样本示例（每个示例包含完整推理链）替换原始 few-shot prompt
- 结论：准确率提升至 56%，提升幅度与模型规模正相关

**步骤 3：控制变量**
- 子论点：提升来自推理结构本身，而非示例数量的增加
- 方法：对比"相同数量示例但无推理链"的消融组
- 结论：无推理链的对照组提升幅度不超过 3%，排除了示例数量的干扰
```

写完论证过程后，自问：**如果我只看这几段，能不能理解作者为什么有资格得出核心论点？** 如果不能，说明某个步骤写得太浅，需要补充。

---

### 研究贡献

说明这篇论文相对于已有工作推进了什么。贡献通常属于以下几类：

- **新方法**：提出了之前没有的技术或框架
- **新发现**：发现了之前未被观察到的现象或规律
- **新视角**：对已知现象提出了新的解释框架
- **新基准**：建立了新的评估体系或数据集

每条贡献写一到两句，说清楚"比之前好在哪里"或"填补了什么空白"。不要只罗列论文自称的贡献，要结合论证过程判断哪些是实质性的。

---

### 局限与反驳

诚实列出论证的边界条件和潜在薄弱处。来源可以是：

- 论文 limitation 节的自我陈述（但要判断作者是否低估了某些局限）
- 实验设计上的问题：数据集偏差、评测指标选择、样本量
- 论证逻辑上的跳跃：某个步骤的结论是否支撑了下一步的前提
- 泛化性问题：结论在哪些条件下成立，超出这些条件时是否仍然有效

不要为了显得公正而罗列无关紧要的局限。只写真正影响核心论点可信度的内容。

---

### 相关工作

列出与本文直接对话的关键先行工作，说明：

- 这篇论文继承了什么（站在谁的肩膀上）
- 这篇论文在哪里与之前的工作分道扬镳

不要堆砌引用列表，只保留理解本文论点所必需的背景。

---

### 参考

- 论文原文链接（arXiv / 期刊页面）
- 代码仓库（如有）
- 相关博客或解读文章（如有）

## 质量检查

- [ ] 核心论点是可验证的命题，不是话题描述
- [ ] 论证过程按逻辑顺序展开，每步有子论点、方法、结论
- [ ] 贡献说明了相对于先行工作的增量
- [ ] 局限聚焦于影响核心论点可信度的内容
- [ ] 读完笔记，无需读原文即可判断"这篇论文可不可信"
