# 调研类文档指引

面向公司、产品、人物的外部信息收集与整理。目标是建立可长期维护的调研档案，而不是一次性报告。

## 适用场景

- 收集整理 AI 公司/初创企业的信息
- 产品竞品分析
- 创始人/关键人物背景调研
- 行业赛道梳理

## 信息收集流程

### 第一阶段：基础信息收集

优先执行，建立基本轮廓：

- 官网 + 官方博客
- 36氪、晚点LatePost、TechCrunch 最新报道
- Crunchbase / IT桔子融资信息
- LinkedIn 公司页面

### 第二阶段：深度挖掘

根据第一阶段的信息缺口，有针对性地补充：

**创始人信息**：LinkedIn 个人页面、Twitter/X、"姓名 + 访谈/演讲"、母校校友信息、投资方对创始人的评价

**产品信息**：Product Hunt 发布页面、GitHub（技术产品）、用户评测（知乎、即刻、Reddit）、App Store/Google Play 评论

**数据信息**：财报（上市公司）、第三方统计平台（SimilarWeb 等）、投资方公开评论、招聘信息透露的团队规模

**竞争格局**：行业媒体深度报道、投资机构行业分析、竞品对比文章

### 信息完整度评估

完成基础收集后评估完整度，决定是否需要深度搜索：

**核心信息**（至少需要 5/7）：
- [ ] 创始人完整姓名（中英文）
- [ ] 创始人教育背景或工作经历
- [ ] 具体融资金额
- [ ] 团队规模（具体数字或范围）
- [ ] 公司成立时间
- [ ] 总部地点
- [ ] 核心产品功能

满足以下任意 2 条时，执行深度搜索：
- 创始人信息少于 2 项
- 团队规模未知
- 用户数据完全缺失
- 产品信息少于 3 个功能点
- 核心信息完整度低于 70%

### 信息质量标记

在报告中标注数据可信度：

| 标记 | 含义 |
|------|------|
| ✅ | 多源验证或官方披露 |
| ⚠️ | 单一来源，需谨慎 |
| ❓ | 推测或计算值 |
| `[待补充]` | 信息存在但未找到 |
| `[未披露]` | 信息可能未公开 |

## 高价值信息来源

**中文**：36氪、晚点LatePost、虎嗅、机器之心、量子位、IT桔子、天眼查、企查查、知乎、即刻

**英文**：TechCrunch、The Information、VentureBeat、Crunchbase、PitchBook、Product Hunt、Hacker News

**特定信息最佳来源**：
- 创始人背景 → LinkedIn、Twitter、百度百科
- 融资详情 → Crunchbase、IT桔子、投资方官网公告
- 用户数据 → 第三方统计、财报、媒体报道
- 技术细节 → GitHub、技术博客、论文

## 更新索引

整理完公司信息后，更新 `知识库/初创/初创企业索引.md`：

1. 在对应地区表格中添加新公司（中国 / 海外）
2. 在领域分类中添加
3. 更新创始人背景速览
4. 在更新日志中记录：
   ```
   - **{YYYY-MM-DD}**：新增 [[明星企业/{公司名}|{公司名}]]：{一句话描述}
   ```

## 质量检查

- [ ] 核心信息完整度 ≥ 70%，或已标注缺失原因
- [ ] 所有数据都标注了来源和可信度
- [ ] 不确定信息用 ⚠️ 或 ❓ 标记
- [ ] 索引文件已更新
- [ ] 客观中立，无主观评价
